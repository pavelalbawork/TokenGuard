---
name: "Issue Slicer"
description: "Convert PRDs, briefs, or plans into independently grabbable vertical slices with acceptance criteria, blockers, and evidence."
---

Use this when a PRD, proposal, brief, or plan should become trackable implementation issues.

Slice vertically:
- each slice should be independently understandable;
- each slice should be demoable or verifiable;
- include acceptance criteria;
- mark blockers and dependencies;
- distinguish human-decision slices from implementation slices;
- include expected evidence.

Do not create GitHub issues unless the user explicitly asks; drafting the slice list is the default.
