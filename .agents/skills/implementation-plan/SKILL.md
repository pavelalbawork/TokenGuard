---
name: "Implementation Plan"
description: "Create a coherent implementation plan with sequencing, risks, and compact execution packets when the plan is substantial."
---

Use this for coherent implementation planning when work spans 3+ files, 2+ surfaces, has ambiguous sequencing, or needs phased execution. Keep plan synthesis single-owner by default.

Required output:

- **Objective + acceptance criteria** (1-2 lines).
- **Execution Packets** for substantial plans: objective, owner fit, scope, dependencies, and acceptance evidence.
- **Critical path**: what should happen first and what can wait.
- **Fan-out opportunities**: independent research or execution packets, or "none".
- **Dependencies and sequencing** between packets.
- **Handoff / review gates** where multiple streams converge.

Use the Work Packets skill when the user wants copy-paste-ready packets for multiple tools/models. Use Prompt Contracts only when those packets are being delegated.

Keep it compact. Vertical slices that produce working, testable progress beat exhaustive decomposition. Do not split plan creation by default; split only independent research questions or execution streams.
